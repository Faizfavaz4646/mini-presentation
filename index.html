Roadmap

1. Not Using Semantic HTML

Bad: Using only <div> and <span> everywhere.

Good: Use <header>, <section>, <article>, etc., for better structure.

2. Ignoring Responsive Design

Websites should work on all screen sizes.

Solution: Use media queries, Flexbox, and Grid.

3. Poor File Organization

Keep HTML, CSS, and JavaScript files in separate folders.

Use meaningful file names.

4. Not Understanding the Box Model

Padding, margin, and border affect element layout.

Solution: Learn how the box model works.

5. Overusing Inline CSS & JavaScript

Bad: Writing styles and scripts directly inside HTML.

Good: Use external CSS and JS files for better structure.

6. Forgetting Performance Optimization

Large images slow down websites.

Solution: Compress images, minify CSS/JS.

7. Not Testing in Different Browsers

Websites may look different in Chrome, Firefox, and Edge.

Always test in multiple browsers.

Conclusion

Learn from mistakes and keep improving!

Debug, test, and follow best practices.
